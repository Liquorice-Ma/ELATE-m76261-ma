# ELATE: Elephant Flow-aware Learning-Accelerated Traffic Engineering in Large-Scale LEO Constellation

ELATE is a scalable traffic engineering (TE) framework for large-scale LEO satellite constellations. By exploiting the heavy-tailed distribution of satellite traffic, ELATE decomposes demands into elephant and mice flows, offloading mice to static shortest paths and focusing GNN + Multi-Agent RL optimization on critical elephant flows. Simulation results on a Starlink constellation show that ELATE reduces MLU by 8–29% over baselines while yielding 11–3288× computational speedups.

Built upon [Teal](https://dl.acm.org/doi/10.1145/3603269.3604857) (ACM SIGCOMM '23).

## Installation

```bash
conda env create -f environment.yml
conda activate teal
pip install -r requirements.txt

# PyTorch + extensions (example for CUDA 11.8)
pip install torch==2.0.0+cu118 -f https://download.pytorch.org/whl/cu118/torch_stable.html
pip install --no-index torch-scatter torch-sparse -f https://data.pyg.org/whl/torch-2.0.0+cu118.html
```

## Usage

### ELATE Mode

```bash
cd run

python teal.py --leo --topo Starlink.json --tm-model toy --lam 5.0 \
    --leo-N 72 --leo-M 22 \
    --epochs 50 --samples 5 \
    --slice-train-start 0 --slice-train-stop 80 \
    --slice-val-start 80 --slice-val-stop 90 \
    --slice-test-start 90 --slice-test-stop 100
```

### TEAL Mode (backward compatible)

```bash
cd run
python teal.py --obj total_flow --topo B4.json --epochs 3 --admm-steps 2
```

### Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `--leo` | off | Enable ELATE mode (elephant/mice separation + counterfactual MARL) |
| `--topo` | required | Topology JSON file (placed in `topologies/`) |
| `--tm-model` | real | Subfolder name under `traffic-matrices/` |
| `--lam` | 5.0 | λ in threshold τ = μ + λσ for elephant/mice separation |
| `--leo-N` | auto | Number of orbits in the LEO grid (e.g. 72 for Starlink) |
| `--leo-M` | auto | Number of satellites per orbit (e.g. 22 for Starlink) |
| `--samples` | 5 | Number of counterfactual samples N_s per agent |
| `--epochs` | 0 | Number of training epochs |
| `--layers` | 6 | Number of GNN layers |
| `--lr` | 0.0001 | Learning rate |
| `--bsz` | 20 | Batch size |
| `--admm-steps` | 5 | ADMM steps (TEAL mode only) |

## Input Format

### Topology

Standard JSON file describing the network graph, placed in `topologies/`. Each edge must include a `capacity` attribute. Example structure:

```json
{
  "directed": true,
  "multigraph": false,
  "nodes": [{"id": 0}, {"id": 1}, ...],
  "links": [{"source": 0, "target": 1, "capacity": 1.0}, ...]
}
```

For LEO constellations, the topology represents the +Grid ISL connectivity (intra-orbit and inter-orbit links).

### Traffic Matrices

Pickle files containing NumPy arrays of shape `[|V|, |V|]`, placed in `traffic-matrices/<tm-model>/`. Naming convention:

```
<topo>_<model>_<seed>_<scale>_traffic-matrix.pkl
```

For example, for a Starlink topology with 1584 nodes:
```
Starlink.json_toy_0_1.0_traffic-matrix.pkl
Starlink.json_toy_1_1.0_traffic-matrix.pkl
...
Starlink.json_toy_99_1.0_traffic-matrix.pkl
```

**Note:** The traffic matrices used in our paper (generated by the random Gravity model with GDP/POP-based city selection probabilities on the Starlink Phase-I Shell-I constellation) are not publicly available. Users need to prepare their own topology and traffic matrix data following the format above.

## Code Structure

```
lib/
├── teal_env.py             # Core environment (ELATE + TEAL modes)
├── FlowGNN.py              # Bipartite GNN with augmented bi-adjacency (Eq. 7-8)
├── teal_actor.py           # Shared stochastic policy network
├── teal_model.py           # Training: counterfactual MARL (Eq. 9-10)
├── flow_identification.py  # Elephant/mice separation (Algorithm 1)
├── leo_topology.py         # +Grid LEO topology generator (utility)
├── leo_path_utils.py       # Modular arithmetic path computation (Eq. 5-6)
├── ADMM.py                 # ADMM post-processing (TEAL mode only)
├── path_utils.py           # k-shortest path computation
└── config.py               # Directory configuration
run/
├── teal.py                 # Entry point
└── teal_helper.py          # CLI argument parsing
topologies/                 # Topology JSON files
traffic-matrices/           # Traffic matrix pickle files
    ├── real/
    └── toy/
```
